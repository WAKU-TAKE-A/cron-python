# 開発履歴と課題ポイント (History)

本ドキュメントでは `cron-python` の開発過程において実施した作業と、直面した技術的課題およびその解決策を記録します。

## 1. コア機能の初期実装
**作業内容:**
- `apscheduler` を導入し、`BlockingScheduler` および `CronTrigger` による定期実行機構を構築。
- `subprocess.Popen` を利用した対象スクリプトの実行と、独自の JSON Logging フォーマッタの作成を行いました。
- タイムアウト時には、孫プロセスを含むWindows上のプロセスツリー全体を確実にキルするため、OSネイティブの `taskkill /F /T /PID` を呼び出す機能を組み込みました。

## 2. コマンドライン引数の解析（`argparse`）の課題と克服
**課題:**
当初は、Python標準の `argparse.REMAINDER` を用いて、任意のJSON引数(`{"input": "data.csv"}` など) をすべて吸い上げる設計にしていました。しかし、オプションフラグ（`--once`など）の後に `REMAINDER` を配置すると、オプション自体が正しく解析されずパラメータに吸収されてしまう等の仕様上のバグが存在しました。

**解決策:**
より確実かつ標準的な手法として、`argparse` へ処理を渡す前に `sys.argv.index('--')` を使ってリストを物理的にスライスし、そこから後ろを「対象スクリプト用のJSON引数」、前を「cron-python用のオプション引数」と完全に分離する設計に変更しました。これにより誤動作を完全に排除できました。

## 3. PyInstaller によるビルド時間の肥大化
**課題:**
単一のEXEファイルを作成する `PyInstaller` の実行が、5分以上待機しても完了しない状態に陥りました。
原因は、ユーザー様の実行環境のベース（`C:\WPy312\python-3.12.4.amd64\` / WinPython）に、データサイエンス等に利用する巨大な外部ライブラリ群（Django, PyQt5, lxml, matplotlib, SciPy など）が多数インストールされていたためでした。
`APScheduler` は内部的に `SQLAlchemy` などの JobStore をサポートしており、PyInstaller がこれらの「潜在的にインポートされる可能性のあるモジュール（Hidden imports）」を検知して、システム上の巨大なパッケージをすべて実行ファイル内にアーカイブしようとしたため、処理が長引いていました。

**解決策:**
不要なモジュールを除外（`--exclude`）するのではなく、クリーンな **仮想環境 (`venv`)** を作成し、そこに必要最小限の依存関係（`requirements.txt` のみ）をインストールしてビルドを行う手順に変更しました。その結果、ビルドは数秒で完了し、極めて軽量かつポータブルな `cron_python.exe` の生成に成功しました。

## 4. EXE動作テストとWindows PowerShellのクォーテーション問題
**課題:**
ビルド後、動作テストの一環として PowerShell から `cron_python.exe` に `{"duration": 1}` というJSON文字列を手引き渡しテストを行おうとしましたが、エラーにより正常に終了しませんでした。
原因は、Windows PowerShell および Command Prompt 特有の「コマンドラインパースによるダブルクォーテーション `"` のエスケープ消失問題」でした。CLIを通して渡された `{"duration": 1}` が、子プロセス側では `{duration: 1}` という不正な形式でパースされてしまい、JSONデコードエラーとなっていました。

**解決策:**
Windowsの複雑なクォーテーションエスケープ仕様に振り回されることなく確実にエンドツーエンドテストを行うため、`test_exe.py` という検証専用のPythonスクリプトを別途作成しました。

## 5. ログ出力と引数処理の改善

## v0.9.4 (2026-04-25)
- **破壊的変更**: `--` による引数分離と JSON 文字列の直接指定を廃止。
- **新機能**: 引数指定用の `--param` (`-p`) オプションを追加。複数の引数を安全に渡せるように改善。
- 内部的な引数処理をリスト形式に変更し、エスケープ文字（バックスラッシュ等）に関する問題を解消。

## v0.9.3 (2026-04-24)
- ログの同時出力機能（ファイル保存＋コンソール表示）を追加。
- 起動直後にジョブを1回実行する `--run-on-start` オプションを実装。
- ログローテーション設定（`--log-max-bytes`, `--log-backup-count`）および標準出力強制フラグ（`--log-stdout`, `--log-stderr`）のドキュメント化。
